| 2026-07-22 | M5 LLM providers | Provider-agnostic layer. `LLMProvider` protocol (`complete(prompt, system, **opts) -> str`), a `RetryingProvider` base that owns retry-with-backoff, timeout, and token accounting so each provider implements only one network round trip. `StubProvider` (echo by default, so protected placeholders survive under test; scripted and fail-N modes too), Ollama, and an OpenAI-compatible provider covering Together/Fireworks/DeepInfra/Groq. The one impure step is an injectable `Transport`, so the hosted providers are tested with a fake, no sockets. Selection is config-driven via `get_provider`; concrete classes are imported only inside the factory, never above `app/llm/`. Backoff is jitter-free on purpose so the sleep schedule is assertable. 13 tests, all network-free. 61 total passing. |
| 2026-07-22 | fix: percent unit | Running an offline demo on a real paragraph surfaced a latent M1 bug: the `quantity_unit` regex ended in `\b`, which never matches after `%` (both sides non-word), so "12.4%" masked only "12.4" and left a loose `%` the model could later move or drop. The old test passed only because it checked `"95%" not in masked`, which masking the number alone satisfies. Fixed with a negative lookahead `(?![A-Za-z0-9])`; strengthened the test to assert the whole "95%" is a mapping value with no loose `%`, plus a named regression test. Added `scripts/demo.py` as an offline, token-free smoke test. Protection now 14 tests. 75 total passing. |
| 2026-07-22 | M6 Prompt library | Versioned prompt files under `prompts/` loaded by id and version (`prompts/system/rewrite.v1.md` -> id `system/rewrite` v1). A test guards that few-shot BEFORE/AFTER carry identical token sets, and another bans em dashes and filler words across all prompts.
